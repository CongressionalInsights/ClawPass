# Recovery Model

## Enrollment baseline

Each approver should keep at least:
- 1 primary passkey
- 1 backup passkey path (second device or platform account sync)
- Optional Ledger method(s)

## Failure cases

- Lost phone with passkey
- Browser profile reset
- Wallet extension loss
- Ledger temporarily unavailable

## Recovery actions

1. Use backup passkey path to regain access.
2. Re-enroll Ledger methods if needed.
3. Rotate or remove stale signer entries.
4. Review audit log for unusual approval attempts.

## Operational recommendation

Treat passkey enrollment as mandatory hygiene for high-risk approvers before assigning approval duties.
